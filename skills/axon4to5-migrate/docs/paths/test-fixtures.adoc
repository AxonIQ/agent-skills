= Test Fixtures

Use this as the compact AF4 -> AF5 fixture migration reference.

== Core Replacement

[cols="1,1,2"]
|===
|AF4 |AF5 |Notes

|`AggregateTestFixture`
|`AxonTestFixture`
|Use for event-sourced entity tests.

|`SagaTestFixture`
|No direct automatic replacement
|Route saga decisions through the `saga` recipe.

|`given(...)`, `when(...)`, `expectEvents(...)`
|AF5 fluent fixture equivalents
|Preserve event order and exception expectations.
|===

== Procedure

. Replace the fixture type first.
. Move command/event imports to AF5 packages.
. Rewrite metadata and message accessor calls to AF5 record-style accessors
  when needed.
. Port exception expectations by behavior, not by exact AF4 exception class.
. Run the migrated test in an isolated scope.

== High-Risk Cases

[cols="1,2"]
|===
|Case |Rule

|Creation policy
|Test both absent and existing entity behavior.

|Multi-entity aggregate
|Include child entity files/events in the isolated scope.

|Polymorphic aggregate
|Verify concrete subtype routing.

|Matcher-heavy assertions
|Prefer explicit event objects over AF4 matcher helpers.

|Saga fixture
|Do not auto-port. Record saga decision first.
|===

== Done

The test compiles, exercises the same command/event behavior, and fails for the
same business reasons as before.
