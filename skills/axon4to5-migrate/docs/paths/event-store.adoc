= Event Store

Migration default: keep AF4 aggregate-keyed event storage readable on AF5.
DCB is a separate migration and is out of scope unless the user has explicitly
chosen it outside this skill.

== Engine Choice

[cols="1,1,2"]
|===
|AF4 backend |AF5 default for this skill |Notes

|JPA `JpaEventStorageEngine`
|`AggregateBasedJpaEventStorageEngine`
|Requires user-owned schema/data plan. This skill emits no SQL.

|Axon Server aggregate stream
|`AggregateBasedAxonServerEventStorageEngine`
|Preserves aggregate-keyed events.

|JDBC
|Blocked/deferred
|No drop-in AF5 equivalent.

|Mongo
|Blocked/deferred
|No AF5 `axon-mongo` path.

|custom subclass
|Blocked/manual
|User must port or replace intentionally.
|===

== Spring Boot

Declare an explicit `EventStorageEngine` bean when AF5 starter defaults would
otherwise select the wrong storage semantics. For JPA, ensure AF5 event-entry
entities are scanned.

== Framework Configurer

Register the chosen aggregate-based engine in the existing
`EventSourcingConfigurer` / component registry. Keep the project on direct
configurer APIs.

== Serializer/Converter

Custom AF4 serializers are not silently ported. Surface required converter work
in recipe output and `learnings.md`.

== Done

AF4 storage wiring is gone or intentionally blocked, the AF5 aggregate-based
engine is wired, and any user-owned runtime/schema work is recorded.
