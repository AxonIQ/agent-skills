= Dead-Letter Queue

DLQ migration is mostly a compatibility and ownership decision. This skill
flags DLQ wiring; it does not migrate rows or schemas.

== Rules

* Preserve processor names.
* Do not create SQL/DDL.
* Mongo DLQ/token-store paths are blocked until a supported AF5 option exists.
* Record user-owned schema/runtime work in `learnings.md`.

== What To Inspect

[cols="1,2"]
|===
|Signal |Action

|`sequencedDeadLetterProcessor`
|Route through the event-processor configuration path.

|processor YAML/properties with `.dlq`
|Carry equivalent settings forward when AF5 supports them.

|Mongo token/DLQ classes
|Emit blocker; do not rewrite silently.

|custom dead-letter table/schema
|Flag out-of-band user work.
|===

== Done

Every DLQ-related processor setting is either migrated, explicitly blocked, or
recorded as user-owned stabilization work.
