= Configuration Migration

Keep the existing wiring style:

* Spring Boot stays Spring Boot with beans/configuration enhancers.
* Plain framework config stays on AF5 configurer APIs.

== Mapping

[cols="1,1,2"]
|===
|AF4 shape |AF5 shape |Rule

|`ConfigurerModule`
|`ConfigurationEnhancer`
|Spring Boot path for non-Axon component customization.

|`DefaultConfigurer.defaultConfiguration()`
|`MessagingConfigurer` / `EventSourcingConfigurer`
|Plain configurer path.

|`Configuration#eventStore()` / `eventBus()`
|topic-specific AF5 API
|Route to event-storage/event-processor as appropriate.

|`commandBus()` / `queryBus()`
|AF5 command/query APIs
|Use gateway configuration-read addenda.

|free lifecycle hooks
|AF5 lifecycle registration
|Preserve start/shutdown ordering.

|generic `registerComponent(...)`
|typed AF5 registration
|Do not invent registrations when no AF5 target exists.
|===

== Procedure

. Classify the class as read-only, write/configurer, or both.
. Apply the smallest mapping from the table.
. Route topic-specific behavior to the owning recipe.
. Comment unsupported AF4 surfaces with `TODO[AF5 migration: <key>]`.
. Compile the configuration class in an isolated scope.

== Done

No AF4 root configuration reads or generic configurer writes remain unless they
are intentionally blocked and recorded.
