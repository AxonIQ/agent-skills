= Interceptors

AF5 interceptors operate on one message at a time and continue through typed
chains returning `MessageStream<?>`.

== Mapping

[cols="1,1"]
|===
|AF4 |AF5

|`MessageDispatchInterceptor#handle(List)`
|`interceptOnDispatch(message, context, chain)`

|`MessageHandlerInterceptor#handle(UnitOfWork, InterceptorChain)`
|`interceptOnHandle(message, context, chain)`

|`CurrentUnitOfWork.get()`
|method `ProcessingContext`

|`unitOfWork.onCommit/onRollback`
|context lifecycle hooks

|`Configurer.register*Interceptor`
|`MessagingConfigurer.register*Interceptor`
|===

== Registration

Spring Boot auto-discovers interceptor beans by generic message type. Add
`@Order` only when AF4 insertion order mattered.

Framework-config projects must move registration calls to AF5
`MessagingConfigurer` APIs.

== Done

AF4 interceptor interfaces are gone, chain continuation uses AF5 APIs, and any
scoping/order ambiguity is recorded instead of guessed.
