= Aggregates

AF5 models AF4 aggregates as event-sourced entities.

== Core Mapping

[cols="1,1,2"]
|===
|AF4 |AF5 |Rule

|`@Aggregate` / `@AggregateRoot`
|`@EventSourced` / `@EventSourcedEntity`
|Remove AF4 aggregate annotation.

|`@AggregateIdentifier`
|event tag / entity id
|Tag every event needed to route the entity.

|constructor command handler
|entity creator + command handler
|Preserve validation and initial event.

|`AggregateLifecycle.apply`
|`EventAppender`
|Append the same events in the same order.

|`@AggregateMember`
|`@EntityMember`
|Keep child routing explicit.

|`AggregateTestFixture`
|`AxonTestFixture`
|Verify behavior, not only compilation.
|===

== Variants

[cols="1,2"]
|===
|Variant |Extra rule

|simple
|One entity, no member/subtype handling.

|creation policy
|Use tests to preserve create/update semantics.

|multi-entity
|Migrate child members and event tags together.

|polymorphic
|Prefer auto-detected concrete type registration when available.
|===

== Done

Handlers compile on AF5, tests preserve command/event behavior, and the entity
is registered through the pinned wiring style.
