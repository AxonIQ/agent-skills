## 10. Dependencies

### Maven / Gradle Migration

Update project dependencies from Axon Framework 4 to Axon Framework 5. The Maven group ID and artifact IDs
changed; the YAML configuration namespace changed from `axon.serializer` to `axon.converter`.

#### Import Mappings

| AF4 artifact | AF5 artifact |
|---|---|
| `org.axonframework:axon-spring-boot-starter` | `io.axoniq.framework:axoniq-spring-boot-starter` |
| `org.axonframework:axon-test` (scope=test) | `org.axonframework.extensions.spring:axon-spring-boot-starter-test` |
| `io.axoniq.console:console-framework-client-spring-boot-starter` | Remove — incompatible with AF5 |

#### Detection

```bash
grep -rn 'org.axonframework' --include='pom.xml' --include='build.gradle' --include='build.gradle.kts' .
grep -rn 'axon.serializer' --include='*.yaml' --include='*.properties' .
```

#### Axon Framework 4 — pom.xml

```xml
<properties>
    <axon.version>4.13.1</axon.version>
</properties>
<dependencies>
    <dependency>
        <groupId>org.axonframework</groupId>
        <artifactId>axon-spring-boot-starter</artifactId>
        <version>${axon.version}</version>
    </dependency>
    <dependency>
        <groupId>org.axonframework</groupId>
        <artifactId>axon-test</artifactId>
        <version>${axon.version}</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

#### Axon Framework 5 — pom.xml

```xml
<properties>
    <axon.version>5.0.0</axon.version>
</properties>
<dependencies>
    <dependency>
        <groupId>io.axoniq.framework</groupId>
        <artifactId>axoniq-spring-boot-starter</artifactId>
        <version>${axon.version}</version>
    </dependency>
    <dependency>
        <groupId>org.axonframework.extensions.spring</groupId>
        <artifactId>axon-spring-boot-starter-test</artifactId>
        <version>${axon.version}</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

#### YAML configuration namespace

```yaml
# AF4
axon:
  serializer:
    general: jackson
    events: jackson

# AF5
axon:
  converter:
    general: jackson
    events: jackson
```

- `axon.serializer.*` → `axon.converter.*` — rename the top-level key only; child keys are unchanged.
- Remove `io.axoniq.console:console-framework-client-spring-boot-starter` — no AF5-compatible version exists yet.

## 20. Aggregates

### Aggregate Class Stereotype

AF4 marked aggregates with `@Aggregate` (Spring) or `@AggregateRoot` (native) plus a `@AggregateIdentifier`
field. AF5 replaces these with `@EventSourced`/`@EventSourcedEntity` with explicit `tagKey` and `idType` attributes.

#### Import Mappings

| AF4 | AF5 (Spring) | AF5 (native) |
|-----|-------------|--------------|
| `org.axonframework.spring.stereotype.Aggregate` | `org.axonframework.extension.spring.stereotype.EventSourced` | — |
| `org.axonframework.modelling.command.AggregateRoot` | — | `org.axonframework.eventsourcing.annotation.EventSourcedEntity` |
| `org.axonframework.modelling.command.AggregateIdentifier` | *(remove — no replacement)* | *(remove)* |

#### Detection

```bash
grep -rn '@Aggregate\|@AggregateRoot\|@AggregateIdentifier' \
  --include='*.java' --include='*.kt' --include='*.scala' .
```

#### Axon Framework 4 Code

```java
import org.axonframework.spring.stereotype.Aggregate;
import org.axonframework.modelling.command.AggregateIdentifier;

@Aggregate
public class Order {
    @AggregateIdentifier
    private OrderId orderId;

    protected Order() { }
}
```

#### Axon Framework 5 Code — Spring Boot

```java
import org.axonframework.extension.spring.stereotype.EventSourced;
import org.axonframework.eventsourcing.annotation.reflection.EntityCreator;

@EventSourced(tagKey = "Order", idType = OrderId.class)
public class Order {
    @EntityCreator
    protected Order() { }
}
```

#### Required attributes (never omit)

| Attribute | Value | Why |
|-----------|-------|-----|
| `tagKey` | Simple class name string, e.g. `"Order"` | Silently breaks on class rename without this |
| `idType` | Class of the AF4 `@AggregateIdentifier` field, e.g. `OrderId.class` | Default is `String.class`; wrong type → silent failure |

- **`.extension.spring.` infix is mandatory** — `org.axonframework.extension.spring.stereotype.EventSourced`.
- **OpenRewrite Phase 1** sometimes rewrites `@Aggregate` → `@EventSourced` without `tagKey`/`idType` — grep and fix.

### Event Emission in Aggregates

AF4 used the static `AggregateLifecycle.apply(event)`. AF5 injects `EventAppender` as the last parameter of
every `@CommandHandler` method.

#### Import Mappings

| AF4 | AF5 |
|-----|-----|
| `import static org.axonframework.modelling.command.AggregateLifecycle.apply` | *(remove)* |
| `org.axonframework.modelling.command.AggregateLifecycle` | *(remove)* |
| `org.axonframework.commandhandling.CommandHandler` | `org.axonframework.messaging.commandhandling.annotation.CommandHandler` |
| — | `org.axonframework.messaging.eventhandling.gateway.EventAppender` |

#### Detection

```bash
grep -rn 'AggregateLifecycle\.apply\|import.*AggregateLifecycle\|import.*commandhandling\.CommandHandler' \
  --include='*.java' --include='*.kt' --include='*.scala' .
```

#### Axon Framework 4 Code

```java
import static org.axonframework.modelling.command.AggregateLifecycle.apply;
import org.axonframework.commandhandling.CommandHandler;

@CommandHandler
public void handle(ShipOrderCommand cmd) {
    apply(new OrderShippedEvent(this.orderId, cmd.getAddress()));
}
```

#### Axon Framework 5 Code

```java
import org.axonframework.messaging.commandhandling.annotation.CommandHandler;
import org.axonframework.messaging.eventhandling.gateway.EventAppender;

@CommandHandler
public void handle(ShipOrderCommand cmd, EventAppender eventAppender) {
    eventAppender.append(new OrderShippedEvent(this.orderId, cmd.getAddress()));
}
```

- **Every `@CommandHandler`** gets `EventAppender` as last parameter — including static factory handlers.
- **`.messaging.` infix is mandatory** for both `@CommandHandler` and `EventAppender`.
- **Child entity `@CommandHandler` methods** also need `EventAppender`.
- **grep after**: any surviving `AggregateLifecycle` call is a compile error.

---

### @EntityCreator — No-Arg Constructor Annotation

AF5 requires the no-arg constructor to be annotated `@EntityCreator`.

#### Import Mappings

| AF4 | AF5 |
|-----|-----|
| *(no annotation)* | `org.axonframework.eventsourcing.annotation.reflection.EntityCreator` |

#### Axon Framework 4 Code

```java
protected Order() { }  // required, no annotation
```

#### Axon Framework 5 Code

```java
import org.axonframework.eventsourcing.annotation.reflection.EntityCreator;

@EntityCreator
protected Order() { }
```

- **`.reflection.` infix is mandatory** — `org.axonframework.eventsourcing.annotation.reflection.EntityCreator`.
- Required on **both** aggregate class and every `@EntityMember` child entity.

---

### Event Class Annotations

AF5 requires `@Event` on every event class. `@EventTag` marks the aggregate routing field. `@Revision` becomes
`@Event(version = N)`.

#### Import Mappings

| AF4 | AF5 |
|-----|-----|
| *(no class annotation)* | `org.axonframework.messaging.eventhandling.annotation.Event` |
| *(no routing field annotation)* | `org.axonframework.eventsourcing.annotation.EventTag` |
| `@Revision("N")` | `@Event(version = N)` |

#### Axon Framework 4 Code

```java
public record OrderShippedEvent(String orderId, String address) { }
```

#### Axon Framework 5 Code

```java
import org.axonframework.messaging.eventhandling.annotation.Event;
import org.axonframework.eventsourcing.annotation.EventTag;

@Event
public record OrderShippedEvent(
    @EventTag(key = "Order")
    String orderId,
    String address
) { }
```

- `@Event` is required — events without it are not recognized by AF5 handlers.
- `@EventTag` `key` must match `tagKey` on the aggregate's `@EventSourced` annotation.
- Every event routed to an aggregate must have `@EventTag` on the routing field.

---

## 30. Event Handlers

### @ProcessingGroup → @Namespace

AF4 grouped event handlers with `@ProcessingGroup("name")`. AF5 renames this to `@Namespace("name")`.

#### Import Mappings

| AF4 | AF5 |
|-----|-----|
| `org.axonframework.config.ProcessingGroup` | `org.axonframework.messaging.core.annotation.Namespace` |

#### Detection

```bash
grep -rn 'import org\.axonframework\.config\.ProcessingGroup\|@ProcessingGroup' \
  --include='*.java' --include='*.kt' --include='*.scala' .
```

#### Axon Framework 4 Code

```java
@ProcessingGroup("orders")
@Component
public class OrderProjector { }
```

#### Axon Framework 5 Code

```java
@Namespace("orders")
@Component
public class OrderProjector { }
```

- After renaming, grep for the old string value to find all external references (YAML, processor definitions).
- A namespace mismatch silently drops events — no compile-time signal.

### Sequencing Policy Migration

AF4 wired sequencing policies in YAML or as `@Bean`. AF5 uses a class-level `@SequencingPolicy` annotation.

#### Import Mappings

| AF4 | AF5 |
|-----|-----|
| YAML `sequencing-policy: beanName` | `@SequencingPolicy` annotation on handler class |
| `org.axonframework.eventhandling.async.SequencingPolicy` | `org.axonframework.messaging.core.sequencing.SequencingPolicy` |
| — | `org.axonframework.messaging.core.annotation.SequencingPolicy` (the annotation) |

#### Detection

```bash
grep -rn 'SequencingPolicy\|sequencing-policy' \
  --include='*.java' --include='*.kt' --include='*.scala' --include='*.yaml' --include='*.properties' .
```

#### Axon Framework 4 (YAML + @Bean)

```yaml
axon:
  eventhandling:
    processors:
      orders:
        sequencing-policy: gameIdSequencingPolicy
```

```java
@Bean
public SequencingPolicy<EventMessage<?>> gameIdSequencingPolicy() {
    return e -> e.getMetaData().get("gameId");
}
```

#### Axon Framework 5 (annotation on handler class)

```java
import org.axonframework.messaging.core.annotation.Namespace;
import org.axonframework.messaging.core.annotation.SequencingPolicy;
import org.axonframework.messaging.core.sequencing.MetadataSequencingPolicy;

@Namespace("orders")
@SequencingPolicy(type = MetadataSequencingPolicy.class, parameters = "gameId")
public class OrderProjector { … }
```

- Remove the YAML `sequencing-policy` key.
- **`@SequencingPolicy` annotation package is `core.annotation`** — not `core.sequencing.annotation`.
- Built-ins: `MetadataSequencingPolicy`, `SequentialPerAggregatePolicy`, `SequentialPolicy`, `FullConcurrencyPolicy`.

#### Custom SequencingPolicy interface (when implements is present)

```java
// AF4
public class MyPolicy implements SequencingPolicy<EventMessage<?>> {
    @Override
    public Object getSequenceIdentifierFor(EventMessage<?> event) { … }
}

// AF5
public class MyPolicy implements SequencingPolicy {
    @Override
    public Optional<Object> sequenceIdentifierFor(EventMessage<?> message, ProcessingContext context) { … }
}
```

### In-Handler Command Dispatch — CommandGateway → CommandDispatcher

AF4 injected `CommandGateway` as a field in event processors. AF5 injects `CommandDispatcher` as a **method
parameter** — async API.

#### Import Mappings

| AF4 | AF5 |
|-----|-----|
| `org.axonframework.commandhandling.gateway.CommandGateway` (field) | *(remove field and constructor injection)* |
| — | `org.axonframework.messaging.commandhandling.gateway.CommandDispatcher` (method param) |
| — | `java.util.concurrent.CompletableFuture` |

#### Detection

```bash
grep -rln 'CommandGateway' --include='*.java' --include='*.kt' --include='*.scala' . \
  | xargs grep -l '@EventHandler'
```

#### Axon Framework 4 Code

```java
@ProcessingGroup("orders")
@Component
public class OrderAutomation {
    private final CommandGateway commandGateway;

    public OrderAutomation(CommandGateway commandGateway) {
        this.commandGateway = commandGateway;
    }

    @EventHandler
    public void on(PaymentReceivedEvent event) {
        commandGateway.sendAndWait(new ShipOrderCommand(event.orderId()));
    }
}
```

#### Axon Framework 5 Code

```java
@Namespace("orders")
@Component
public class OrderAutomation {
    // No CommandGateway field

    @EventHandler
    public CompletableFuture<?> on(PaymentReceivedEvent event, CommandDispatcher commandDispatcher) {
        return commandDispatcher.send(new ShipOrderCommand(event.orderId())).getResultMessage();
    }
}
```

#### Key API differences

| AF4 | AF5 |
|-----|-----|
| `commandGateway.sendAndWait(cmd)` | `commandDispatcher.send(cmd).getResultMessage()` |
| `commandGateway.sendAndWait(cmd, meta)` | `commandDispatcher.send(cmd, meta).getResultMessage()` |
| try/catch | `.exceptionallyCompose(err -> …)` |
| `void` handler return | `CompletableFuture<?>` handler return |

- Remove `CommandGateway` field and constructor injection entirely.
- Handler must return `CompletableFuture<?>` — do not call `.join()` inside the handler.

## 50. Interceptors

### MessageHandlerInterceptor — Handle Method Signature Migration

AF4 handler interceptors used `handle(UnitOfWork, InterceptorChain)`. AF5 uses
`interceptOnHandle(M, ProcessingContext, MessageHandlerInterceptorChain<M>)`.

#### Import Mappings

| AF4 | AF5 |
|-----|-----|
| `org.axonframework.messaging.unitofwork.UnitOfWork` | *(remove)* |
| `org.axonframework.messaging.InterceptorChain` | *(remove)* |
| `org.axonframework.messaging.MessageHandlerInterceptor` | `org.axonframework.messaging.core.MessageHandlerInterceptor` |
| — | `org.axonframework.messaging.core.MessageHandlerInterceptorChain` |
| — | `org.axonframework.messaging.core.MessageStream` |
| — | `org.axonframework.messaging.core.unitofwork.ProcessingContext` |

#### Detection

```bash
grep -rn 'implements MessageHandlerInterceptor\|UnitOfWork.*InterceptorChain' \
  --include='*.java' --include='*.kt' --include='*.scala' .
```

#### Axon Framework 4 Code

```java
public class AuthInterceptor implements MessageHandlerInterceptor<CommandMessage<?>> {
    @Override
    public Object handle(
            UnitOfWork<? extends CommandMessage<?>> unitOfWork,
            InterceptorChain interceptorChain) throws Exception {
        CommandMessage<?> cmd = unitOfWork.getMessage();
        validate(cmd.getMetaData().get("playerId"));
        return interceptorChain.proceed();
    }
}
```

#### Axon Framework 5 Code

```java
public class AuthInterceptor implements MessageHandlerInterceptor<CommandMessage> {
    @Override
    public MessageStream<?> interceptOnHandle(
            CommandMessage message,
            ProcessingContext context,
            MessageHandlerInterceptorChain<CommandMessage> chain) {
        validate(message.metaData().get("playerId"));
        return chain.proceed(message, context);
    }
}
```

#### Key changes

| Element | AF4 | AF5 |
|---------|-----|-----|
| Method name | `handle` | `interceptOnHandle` |
| Parameters | `UnitOfWork<? extends M>, InterceptorChain` | `M, ProcessingContext, MessageHandlerInterceptorChain<M>` |
| Return type | `Object` | `MessageStream<?>` |
| Chain call | `interceptorChain.proceed()` | `chain.proceed(message, context)` |
| Generic | `CommandMessage<?>` | `CommandMessage` (no wildcard) |
| Checked exceptions | `throws Exception` | removed |

#### Dispatching commands from interceptors (AF5)

```java
// AF4 — loading aggregate via Repository
repository.loadOrCreate(id, () -> new ResourcesPool(id))
          .execute(rp -> rp.handle(cmd));

// AF5 — use CommandDispatcher bound to current context
CommandDispatcher.forContext(context)
    .send(withdrawCommand)
    .resultAs(Void.class)
    .join();
```

#### UnitOfWork lifecycle hooks → ProcessingContext

| AF4 | AF5 |
|-----|-----|
| `unitOfWork.getMessage()` | use `message` parameter directly |
| `unitOfWork.onCommit(uow -> {…})` | `context.runOnAfterCommit(ctx -> {…})` |
| `unitOfWork.onRollback(uow -> {…})` | `context.onError((ctx, err) -> {…})` |
| `unitOfWork.onCleanup(uow -> {…})` | `context.doFinally(ctx -> {…})` |

## 60. Sagas

### Saga Migration — @Saga → @Component @DisallowReplay

AF5 sagas are regular Spring `@Component` + JPA entities. The framework no longer manages saga state.

#### Import Mappings

| AF4 | AF5 |
|-----|-----|
| `org.axonframework.spring.stereotype.Saga` | *(remove)* |
| `org.axonframework.modelling.saga.SagaEventHandler` | `org.axonframework.messaging.eventhandling.annotation.EventHandler` |
| `org.axonframework.modelling.saga.StartSaga` | *(remove)* |
| `org.axonframework.modelling.saga.EndSaga` | *(remove)* |
| `org.axonframework.modelling.saga.SagaLifecycle` | *(remove)* |

#### Detection

```bash
grep -rn '@Saga\|import.*stereotype.Saga\|import.*modelling.saga' \
  --include='*.java' --include='*.kt' --include='*.scala' .
```

#### Axon Framework 4 Code

```java
@Saga
public class OrderFulfillmentSaga {
    @SagaEventHandler(associationProperty = "orderId")
    @StartSaga
    public void on(OrderPlacedEvent event) {
        SagaLifecycle.associateWith("orderId", event.orderId());
    }
}
```

#### Axon Framework 5 Code

```java
@Entity
@Table(name = "order_fulfillment_saga")
@Component
@DisallowReplay
public class OrderFulfillmentSaga {
    @Id
    private String orderId;
    private String status;

    protected OrderFulfillmentSaga() { }

    @EventHandler
    public CompletableFuture<?> on(OrderPlacedEvent event, CommandDispatcher dispatcher) {
        this.orderId = event.orderId();
        this.status = "STARTED";
        // persist via repository...
        return dispatcher.send(new ProcessPaymentCommand(event.orderId())).getResultMessage();
    }
}
```

- Annotate with `@Entity` + `@Table` + add `@Id` field + protected no-arg constructor.
- Persist state manually using a Spring Data repository.
- `@SagaEventHandler(associationProperty)` → `@EventHandler`; load state from JPA by correlation field.
- `SagaLifecycle.*` removed — no equivalent.
- **Deadline Manager (`@DeadlineHandler`)** — no AF5 equivalent; blocker if used.

## 70. Event Store

### Event Store Configuration — JPA

AF5 requires an explicit `EventStorageEngine` bean when using JPA (Axon Server disabled).

#### Detection

```bash
grep -rn 'EventStorageEngine\|EmbeddedEventStore\|JpaEventStorageEngine' \
  --include='*.java' --include='*.kt' --include='*.scala' .
```

#### Axon Framework 4

```java
// Auto-configured — no explicit bean needed
```

#### Axon Framework 5

```java
import org.axonframework.eventsourcing.jpa.AggregateBasedJpaEventStorageEngine;
import org.axonframework.eventsourcing.jpa.JpaTransactionalExecutorProvider;
import org.axonframework.eventsourcing.jpa.EventConverter;

@Configuration
@EntityScan(basePackages = {"com.example", "org.axonframework", "io.axoniq.framework"})
@ConditionalOnProperty(name = "axon.axonserver.enabled", havingValue = "false")
public class EventStoreConfiguration {

    @Bean
    public EventStorageEngine eventStorageEngine(EntityManagerFactory emf, EventConverter eventConverter) {
        return new AggregateBasedJpaEventStorageEngine(
            new JpaTransactionalExecutorProvider(emf),
            eventConverter,
            UnaryOperator.identity()
        );
    }
}
```

- Add `@EntityScan` with `org.axonframework` and `io.axoniq.framework` packages.
- `@ConditionalOnProperty` guards it to JPA-only deployments.

## 80. Tests

### AggregateTestFixture → AxonTestFixture

AF4 used `AggregateTestFixture<T>`. AF5 replaces it with `AxonTestFixture` wrapping `EventSourcingConfigurer`.

#### Import Mappings

| AF4 | AF5 |
|-----|-----|
| `org.axonframework.test.aggregate.AggregateTestFixture` | `org.axonframework.test.fixture.AxonTestFixture` |
| — | `org.axonframework.eventsourcing.EventSourcingConfigurer` |
| — | `org.axonframework.eventsourcing.modules.EventSourcedEntityModule` |

#### Detection

```bash
grep -rn 'AggregateTestFixture' --include='*.java' --include='*.kt' --include='*.scala' .
```

#### Axon Framework 4 Code

```java
private AggregateTestFixture<Order> fixture;

@BeforeEach
void setUp() {
    fixture = new AggregateTestFixture<>(Order.class);
}

@Test
void shouldShipOrder() {
    fixture.given(new OrderCreatedEvent("o1"))
           .when(new ShipOrderCommand("o1", "123 Main St"))
           .expectEvents(new OrderShippedEvent("o1", "123 Main St"));
}
```

#### Axon Framework 5 Code

```java
private AxonTestFixture fixture;

@BeforeEach
void setUp() {
    fixture = AxonTestFixture.with(
        EventSourcingConfigurer.create()
            .registerEntity(EventSourcedEntityModule.autodetected(OrderId.class, Order.class))
    );
}

@AfterEach
void tearDown() {
    fixture.stop();
}

@Test
void shouldShipOrder() {
    fixture.given().events(new OrderCreatedEvent("o1"))
           .when().command(new ShipOrderCommand("o1", "123 Main St"))
           .then().events(new OrderShippedEvent("o1", "123 Main St"));
}
```

#### Fluent DSL changes

| AF4 | AF5 |
